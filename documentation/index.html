Getting started

Introduction

Welcome to BlazorStore. A beautifully crafted Mobile PWA & Site Template made to run incredibly fast and be extremely easy to edit and customize.

Based on MudBlazor with an emphasis on ease of use and clear structure. It is perfect for .NET developers who want to rapidly build web applications without having to struggle with CSS and Javascript

In BlazorStore, we added PWA feature, with an E-Commerce app material template that looks and behaves as if it is a mobile app. PWAs are built to take advantage of native mobile device features, without requiring the end user to visit an app store, make a purchase and download software locally

Running Applications

Once you have downloaded the files, all you have to do is include the following files in your project:

Letâ€™s open up the BlazorStore.sln solution file. Wait for Visual Studio to finish installing all the required packages. You can see the Folder Structure of the Project below.

Important Note

Please make sure that the BlazorStore.Server project is set as the startup project.

By default, the localdb instance of your SQLServer is used as the Data Source with your namespace as the DB Name. Feel free to change the connection strings which are located at the appsettings.json and appsettings.Development.json in the Server Project. 

"ConnectionStrings": {"DefaultConnection": "Data Source=(localdb)\\mssqllocaldb;Initial Catalog=BlazorStore;Integrated Security=True;MultipleActiveResultSets=True" },

Once you have fixed up your connection string, the next step is to create the database and update it. For this, open up the package manager console and set the default project as the BlazorStore.Insfrastructure project. Make sure that the BlazorStore.Server is set as the startup project as well. This is because all the packages and DBContexts related to building / migrating EFCore SQL Server databases are installed in the Infrastructure project.

With that done, run the following command.

update-database

Important Note

BlazorStore have already added the latest migrations of the database. You are no need to run the add-migration command again.

Once the database migration is done, you will be able to see the newly created database in the server explorer. Default Data like Roles and Users will be seeded into the database when you run the application for the first time.

With the database migrations done, we are good to go. We recommend using the Kestrel Server instead of the IIS Server Variant. To switch to Kestrel, do the following.

Once you run the application with Kestrel, you can see the below terminal popup. Here you can note that some default data is seeded while the application is run for the first time. This includes default roles, users, and permissions.

Docker Support

By default, Docker project build is disabled for BlazorStore. You can re-enable them by right-clicking the Solution in Visual Studio and enabling â€˜docker-composeâ€™ Build option in the Configuration Properties section!

Here are the steps:

  • Install Docker on Windows via https://docs.docker.com/docker-for-windows/install/
  • Open up Powershell on Windows and run the following
    • cd c:\
    • dotnet dev-certs https -ep $env:USERPROFILE\.aspnet\https\aspnetapp.pfx -p securePassword123
    • dotnet dev-certs https --trust
    • Note â€“ Make sure that you use the same password that has been configured in the docker-compose.yml file. By default, securePassword123 is configured.
  • 5005 & 5006 are the ports set up to run BlazorStore on Docker, so make sure that these ports are free. You could also change the ports in the docker-compose.yml and Server\Dockerfile files.
  • Now navigate back to the root of the BlazorStore Project on your local machine and run the following via terminal â€“ docker-compose -f â€˜docker-compose.ymlâ€™ up â€“build
  • This will start pulling MSSQL Server Image from Docker Hub if you donâ€™t already have this image. Itâ€™s around 500+ Mbs of download.
  • Once that is done, dotnet SDKs and runtimes are downloaded, if not present already. Thatâ€™s almost 200+ more Mbs of download.
  • PS If you find any issues while Docker installs the NuGet packages, it is most likely that your SSL certificates are not installed properly. Apart from that I also added the â€“disable-parallel in the Server\Dockerfileto ensure network issues donâ€™t pop up. You can remove this option to speed up the build process.
  • Thatâ€™s almost everything. Once the containers are available, migrations are updated in the MSSQL DB, default data is seeded.
  • Browse to https://localhost:5005/ to use your version of BlazorStore!

User Registration

If you are logged in as an Administrator, you get permissions to add new users to the system. Letâ€™s add one!

Important Note

Make sure that the username you enter is 6 characters or longer.

Click Register after filling in the details. You can notice the two checkboxes.

  • Activate User â€“ This system has additional security that even after registration, the user should be approved by the administrator to access the resource. Unless the user is active, he/she will be locked out of the system. To skip this, check the Activate User box.
  • Auto-Confirm Email â€“ To register a new account with the system, the user has to confirm his/her email. BlazorHero sends an activation mail to the registered Email Id. In our case, by default, the registration mail goes to the ethereal mailbox. Again, to skip the part where the email has to confirmed, just check the Auto-Confirm Email checkbox.

Once the user is registered, an Email gets scheduled as a Background Job via Hangfire which then sends it to the Ethereal Mailbox.

Apart from this, users can also anonymously register into the application by navigating to the /register page.

Publishing Applications

Publish Application to IIS

Important note

Make sure that you have installed Net 5.0 SDK in your IIS Server Download

  • Step 1. Select BlazorStore.Server as active (startup) project Publish
  • Step 2. Target: choose Folder
  • Step 3. Location: Select your IIS web destination folder - please make sure you open VS under admin to have write privilege to C disk or you can publish to another local disk D then copy it to IIS blazorstore root.
  • Step 4. Settings: Target net5.0, Deployment Mode Framework-dependent, Target Runtime Portable
  • Step 5. Click Publish button to start publishing
  • Step 6. Once publish done, go to published root folder, create new Files folder, as below image:

Publish Done.
